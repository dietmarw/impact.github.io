Nuances

Cannot elaborate the dependency tree until we pick a particular version of a library

Order of child dependencies matters (e.g., order in uses annotation)

Constraints

Currently, we take the constraints specified by a library literally (i.e., the current definition of the uses annotation specifies that each version of a library can only be compatible with a single version of any other library)

It would be good if, in the future, Modelica allowed multiple candidates to be used via the uses annotation.

Eventually, we'd like to move to a more "rational" approach where semantic versioning was taken into account when determining constraints.
